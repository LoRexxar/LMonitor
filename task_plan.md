# SimC 功能优化计划

## 目标
- 完成你提出的 7 项 SimC 需求：配置专精字段、模板专精字段与匹配（支持多模板同时启用并按专精匹配）、配置筛选、常规模拟可选 time/目标数、属性模拟步长可配置、天赋预览图、APL 翻译预览。
- 保持现有 SimC 任务流程可用，避免影响已存在的属性模拟与结果展示。

## 范围
- 后端：`botend/models.py`、`botend/dashboard/api.py`、`botend/controller/plugins/simc/SimcMonitor.py`
- 前端：`templates/dashboard/index.html`、`static/dashboard/js/main.js`
- 模板：`LMonitor/simc_template.txt`（或数据库 `SimcTemplate` 激活模板）
- 迁移：`botend/migrations/` 新增 migration

## 分阶段执行

### Phase 1: 数据模型与迁移（专精字段）
**状态**: in_progress
- 在 `SimcProfile` 增加专精字段（建议 `spec`，并预留默认值）。
- 补充迁移脚本，兼容历史数据（默认值 + 非破坏性变更）。
- 更新 API 序列化与反序列化，确保创建/更新/复制配置都带上专精字段。
- 统一模板专精占位符（如 `{spec}`）并建立与 `SimcProfile.spec` 的匹配替换逻辑。
- 对历史模板兼容：当模板未包含 `{spec}` 时，执行前自动替换或注入 `spec=` 行。
- 取消“模板只能单启用”约束：允许多个模板 `is_active=True` 并行存在。
- 执行时按 `SimcProfile.spec` 匹配已启用模板；无匹配时按明确兜底策略返回错误或回退默认模板。

**验收标准**
- 新建/编辑/复制 `SimcProfile` 时可读写 `spec`。
- 生成的 simc 文本中 `spec` 与配置专精一致，不再固定为单一专精。
- 同时启用多个模板时，任务会命中对应专精模板执行，不发生启用冲突。
- 旧配置在不手工修复的情况下可正常读取与模拟。

### Phase 2: 配置管理筛选能力
**状态**: pending
- 为 SimC 配置管理增加前端筛选控件（至少包含：名称关键字、战斗风格、专精）。
- 后端支持按筛选参数返回配置列表（仅当前用户）。
- 与现有搜索逻辑兼容，不破坏分页与已有表格加载。

**验收标准**
- UI 可组合筛选，结果数量与条件一致。
- 清空筛选后恢复完整列表。

### Phase 3: 任务参数入口增强
**状态**: pending
- 在“新增/编辑 SimC 任务（常规模拟）”里增加 time、target_count 可选覆盖项。
- 通过 `SimcTask.ext`（JSON）存储任务级覆盖参数，不污染配置基线。
- `SimcMonitor` 执行时优先读取任务覆盖值，未设置则回退 `SimcProfile` 默认值。
- 在“属性模拟”任务入口增加步长设置（默认 50，可自定义）。
- `SimcMonitor.process_attribute_simulation` 使用任务级步长替代硬编码 50，并做最小值校验。

**验收标准**
- 同一配置可创建多个不同 time/目标数的常规模拟任务。
- 生成的 simc 文本与最终结果体现任务级参数。
- 属性模拟可通过 UI 设置步长，未设置时默认 50，设置后按新步长生成测试点。

### Phase 4: 天赋预览图
**状态**: pending
- 在 SimC 配置区增加“天赋预览”面板（编辑时实时刷新）。
- 实现天赋字符串到预览资源的映射策略：
- 优先：可直接渲染的静态预览图 URL。
- 兜底：显示可点击的外部预览链接 + 校验失败提示。

**验收标准**
- 对合法天赋串能显示预览内容（图或可访问预览链接）。
- 对非法/空天赋串给出明确提示，不阻断保存。

### Phase 5: APL 列表翻译预览
**状态**: pending
- 在已保存 APL 列表中增加“翻译预览”入口。
- 复用现有关键字映射与转换 API，实现原文/翻译并排展示。
- 保留一键复制原文与翻译结果能力。

**验收标准**
- 用户无需离开列表即可查看任意 APL 的翻译预览。
- 翻译内容与 `SimcAplKeywordPair` 当前映射一致。

### Phase 6: 联调与回归
**状态**: pending
- 覆盖关键流程：创建配置、模板匹配、筛选、创建常规模拟、属性模拟步长、生成结果、APL 翻译预览。
- 进行最小回归：属性模拟流程、现有任务列表、配置复制功能。

**验收标准**
- 无新增明显前后端报错。
- 7 项需求均可演示。

## 风险与应对
- 天赋预览图来源不稳定：采用“图像优先 + 链接兜底”。
- `ext` 兼容历史字符串：统一做 JSON 解析容错（字符串/对象双兼容）。
- 模板格式不统一：对 `{spec}` 缺失模板做运行时兼容补丁，避免旧模板失效。
- 多模板命中歧义：增加模板匹配优先级规则（精确专精 > 同职业默认 > 全局默认）。
- 现有前端 JS 体量较大：按模块函数改动，避免全局回归风险。

## 决策记录
- 任务级 time/target_count 使用 `SimcTask.ext` 承载，避免污染配置主表语义。
- 配置层面保留 `SimcProfile.time/target_count` 作为默认值。
- 模板管理允许多启用；执行阶段基于专精路由选择模板，不再强制全局唯一启用模板。
